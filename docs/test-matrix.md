# テストマトリックス（Pet Life Plus）

`docs/09-test-report.html` と `docs/test-checklist.md` をもとに、機能・ロール・品質観点・実行条件の対応関係を一覧化したマトリックスです。

## 1. 目的
- テストケースの抜け漏れを機能横断で確認する
- ロール差異、外部連携、フォールバック、異常系の網羅状況を見える化する
- 実施時に「どの環境・どのアカウントで何を確認するか」を先に固める

## 2. 判定記号
- `○`: 対応あり
- `△`: 条件付きで対応あり
- `-`: 対象外

## 7. 未実施になりやすい観点
| 観点 | 理由 | 代替記録方法 |
|---|---|---|
| SendGrid メール送信 | 検証用メール環境が必要 | 送信ログ、メールキュー、受信画面キャプチャ |
| LINE Broadcast | フォロワーとチャネル設定が必要 | LINE 管理画面ログ、受信端末キャプチャ |
| Slack DM 応答 | Slack App と DM 先が必要 | Slack スクリーンショット、アプリログ |
| Zoom 予約導線 | Premium アカウントと Zoom 設定が必要 | 予約詳細画面、通知、発行リンク確認 |
| API キー未設定フォールバック | 環境変数切替が必要 | 切替日時、構成差分、レスポンス記録 |

## 8. 実施順の推奨
1. 認証・ロール別アクセス制御
2. 基本 CRUD（ユーザー、ペット、健康記録）
3. 予約、通知、カレンダー
4. AI症状チェック、チャットボット
5. 外部連携（LINE、Slack、Zoom、SendGrid）
6. 管理機能、請求、印刷

## 9. 補足
- `△` は条件付き対応です。たとえば STAFF や VET は機能の一部のみ実行できるケースがあります。
- 本マトリックスは観点整理用です。実際の判定記録は `docs/test-checklist.md` に残してください。
- 追加機能が増えた場合は、まず本マトリックスに観点を追加し、その後チェックリストへ具体的な項目を追加する運用を推奨します。
