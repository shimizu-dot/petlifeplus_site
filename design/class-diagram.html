パッケージの役割 必須
各パッケージの責務を説明
パッケージ 役割 使用するアノテーション
config 認証・認可、MyBatis、Webリソース、初期化などの設定 @Configuration, @EnableWebSecurity, @Bean, @Component
controller 画面遷移（Thymeleaf）と操作受付（/app/**, 一部 /api/**） @Controller, @RequestMapping, @GetMapping, @PostMapping, @PatchMapping, @DeleteMapping
controller.line / controller.slack 外部Webhook受信エンドポイント @Controller, @RequestMapping, @PostMapping
service 業務ロジック実装（権限判定、プラン制御、外部API連携を含む） @Service, @Value
service.line / service.slack LINE/Slack送受信・署名検証などの連携処理 @Service
mapper データベースアクセス @Mapper
entity DB行マッピング用のドメインデータ定義（record中心） Java record（アノテーション最小）
dto 画面/API入出力データの受け渡し @Data, @NotNull, @NotBlank, @Size, @Pattern, @DateTimeFormat
exception 業務例外・エラーハンドリング @RestControllerAdvice, @ExceptionHandler

3 Repository（リポジトリ）

PetMapper.java 必須
ペットリポジトリ（MyBatis Mapper）のインターフェース定義

実装準拠ポイント:
`insertReturningId` で採番IDを返却し、更新系は `int` で更新件数を返す。

メソッド 引数 戻り値 SQL種別 説明
findAll limit: int, offset: int List<PetEntity> SELECT 全件取得（管理者）
countAll なし long SELECT 全件数取得（deleted_at IS NULL）
findByOwnerUserId ownerUserId: Long, limit: int, offset: int List<PetEntity> SELECT オーナー別一覧取得
findActiveByOwnerUserId ownerUserId: Long List<PetEntity> SELECT オーナー別の有効ペット取得
countByOwnerUserId ownerUserId: Long long SELECT オーナー別件数取得
findById id: Long PetEntity SELECT ID指定の単件取得
findByIdAndOwnerUserId id: Long, ownerUserId: Long PetEntity SELECT 所有者条件つき単件取得
insertReturningId pet: PetEntity Long INSERT 新規登録して採番ID返却
update pet: PetEntity int UPDATE 更新（件数返却）
softDelete id: Long, deletedAt: LocalDateTime int UPDATE 論理削除（deleted_at）
countLinkedDataFlags petId: Long int SELECT 関連データ存在フラグ件数
markDeceased id: Long, deceasedAt: LocalDateTime int UPDATE 永眠日時を設定（未設定時のみ）

6 クラス間の依存関係

依存関係図 必須
Controller → Service → Repository → Entity の依存関係

実装準拠ポイント：
① Controller は Spring DI（コンストラクタインジェクション）で Service を注入。
② Service は MyBatis @Mapper を通じて PostgreSQL にアクセス（XML なし、@Select/@Insert/@Update/@Delete アノテーション）。
③ 全テーブルは soft-delete（deleted_at IS NULL フィルタ）を適用。
④ 認可制御は Service 層の hasStaffAccess() / PlanAccessService で実施（plan_features テーブル参照）。
⑤ 破線枠（*）は schema 定義済み・Mapper 実装済みだが Controller/Service 未接続（schema-only 状態）。
