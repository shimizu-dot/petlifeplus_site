## テーブル定義（表形式）

方針:
- 全テーブルに `created_at` を付与し、更新が発生するテーブルには `updated_at` を付与
- 認証情報は `users.password_hash VARCHAR(255)`
- 外部キー制約を明記
- 論理削除対象に `deleted_at` を付与

### appointment_slots
SUPER/ADMIN/VET/STAFF が自動生成スロット（9:30–17:00 30分刻み）を調整するためのテーブル。
- `is_blocked = false`（追加枠）: 業務時間外や特別スロットを追加
- `is_blocked = true`（ブロック枠）: 特定の自動生成スロットを非表示化

| カラム名 | データ型 | NULL許可 | デフォルト値 | 制約 | 説明 |

| id | BIGINT | NO | GENERATED BY DEFAULT AS IDENTITY | PK | 枠ID |
| slot_datetime | TIMESTAMP | NO | | | 枠日時 |
| note | VARCHAR(200) | YES | NULL | | 備考（例: 院内研修のためブロック） |
| is_blocked | BOOLEAN | NO | FALSE | | true=ブロック枠 / false=追加枠 |
| created_by_user_id | BIGINT | NO | | FK -> users.id | 作成者ID |
| deleted_at | TIMESTAMP | YES | NULL | | 論理削除日時 |
| created_at | TIMESTAMP | NO | CURRENT_TIMESTAMP | | 作成日時 |
