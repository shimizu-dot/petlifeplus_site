# 要件定義まとめ（Pet Life Plus）

## 1. システム概要
- サービス名: ペットライフプラス（Pet Life Plus）
- 利用者: 管理者、一般ユーザー
- 目的: ペットの健康記録、AI症状チェック、予約、診療・相談履歴を一元管理し、早期異変の発見、受診判断の支援、継続的な健康管理を実現する。

## 2. システム構成
- フロントエンド: HTML / CSS / JavaScript（静的ファイル、`frontend/public/` に配置）
- テンプレートエンジン: Thymeleaf（バックエンド側のサーバーサイドレンダリング）
- バックエンド: Spring Boot（Controller → Service → Repository）
- データベース: PostgreSQL

### F-001
機能名：ログイン認証
機能ID：F-001  機能の詳細仕様を記載：メールアドレス/パスワードで認証し、成功時にセッションを発行してダッシュボードへ遷移する。

画面レイアウト
- メールアドレス入力、パスワード入力、ログインボタン、再設定リンクを配置。

- システム: 認証処理失敗時に汎用エラーメッセージを表示。

### F-012
機能名：通知・リマインド配信
機能ID：F-012  機能の詳細仕様を記載：通知作成、配信対象設定、配信実行、配信履歴管理に加え、スケジュールリマインダーのユーザー確認を行う。

#### 実装済み機能

**配信済み通知一覧 (`GET /app/notifications`)**
- `notifications` + `notification_recipients` テーブルに基づく配信済み通知を表示
- 未読件数をバッジ表示
- 個別既読 (`POST /app/notifications/{id}/read`) / 一括既読 (`POST /app/notifications/read-all`) に対応

**予定リマインダー（動的生成、30日以内）**

| 種別 | バッジ | データソース | リマインダーキー形式 |
| --- | --- | --- | --- |
| 次回ワクチン・ケア予定日 | リマインダー（青） | `pet_care_records.next_due_on` | `CARE:{petId}:{careType}:{nextDueOn}` |
| 診療予約日 | リマインダー（青） | `appointments.scheduled_at` | `APPT:{appointmentId}` |
| サブスクリプション更新日 | 更新お知らせ（黄） | `subscriptions.end_date` | `SUB:{subscriptionId}:{endDate}` |

**リマインダー確認 (`POST /app/notifications/reminders/dismiss`)**
- 「確認」ボタン押下でリマインダーキーを `dismissed_reminders` テーブルに保存
- 次回表示時は確認済みリマインダーを除外

画面レイアウト
- 上段: 予定・お知らせリマインダー（各行に「確認」ボタン）
- 下段: 配信済み通知一覧（各行に「既読」ボタン）

処理フロー（リマインダー確認）
1. 通知ページでリマインダー一覧を表示する。
2. 「確認」ボタン押下で `POST /app/notifications/reminders/dismiss` を送信する。
3. `dismissed_reminders` に `(user_id, reminder_key)` を保存する。
4. 通知ページへリダイレクト、確認済みリマインダーは非表示になる。

入力項目一覧（配信通知）
| 項目名 | データ型 | 必須 | バリデーション | 備考 |
| --- | --- | --- | --- | --- |
| 件名 | String | ○ | 必須チェック / 100文字以内 |  |
| 本文 | String | ○ | 必須チェック / 2000文字以内 |  |
| 配信日時 | DateTime | — | 日時形式 / 現在以降 | 予約配信 |

## 5. 非機能要件

### セキュリティ要件
- 認証方式（Spring Security）: フォームログインを基本にSpring Securityで認証・認可を実装し、URL単位でアクセス制御する。
- パスワードの暗号化方式: BCryptPasswordEncoderでハッシュ化して保存し、平文保存は行わない。
- CSRF対策: Spring SecurityのCSRF保護を有効化し、フォーム送信時にCSRFトークンを検証する。
- XSS対策: 入力値バリデーションと出力時エスケープ（Thymeleaf自動エスケープ）を実施する。
- SQLインジェクション対策: MyBatis Mapperでバインド変数（`#{}`）を使用し、SQL文字列連結（`${}`の安易な利用）を禁止する。
- セッション管理: ログイン時のセッションID再生成、セッションタイムアウト設定、ログアウト時のセッション破棄を実施する。
