あなたはフロントエンドエンジニアです。以下の参照ドキュメントを根拠に、静的Webサイト（HTML/CSS/JavaScript）を実装してください。

【参照ドキュメント】
- 企画提案書: ../../docs/01-proposal.html
- サイトマップ: docs/04-sitemap.html
- ワイヤーフレーム: docs/05-wireframe.html
- デザインガイドライン: ../../docs/06-design-guide.html

【プロジェクト概要】
- サービス名: ペット健康管理プラットホーム（ペットライフプラス）
- ターゲット: 都市部で働く20〜40代の犬飼い主
- 価値提供: 健康記録、AI症状チェック、オンライン診療・訪問ケア導線

【実装要件】
1. レスポンシブ対応（モバイルファースト）
- 基本はスマホ1カラム設計
- 768px以上でレイアウト拡張（カード複数列、ヘッダー導線強化）
- 1200px程度でPC最適化（余白・最大幅を調整）

2. セマンティックHTML
- `header`, `nav`, `main`, `section`, `article`, `aside`, `footer` を適切に使用
- 見出し階層（h1-h3）を正しく構成
- フォーム要素は `label` と `id` を必ず関連付ける

3. アクセシビリティ
- 画像に適切な `alt`
- キーボード操作可能（フォーカス可視化）
- コントラスト比を確保
- `aria-*` は必要箇所のみ適切に使用
- スキップリンク（本文へ移動）を実装
- ナビのハンバーガーは `button` 要素で実装し、開閉状態を `aria-expanded` で管理

4. デザインガイドライン準拠
- 色:
  - メイン: #1D4ED8
  - サブ: #0EA5A4
  - アクセント: #F59E0B
  - 背景: #F8FAFC
  - テキスト: #0F172A / #64748B
- フォント: M PLUS Rounded 1c（見出しは Fredoka、なければ sans-serif）
- 信頼感・伴走感・俊敏性を視覚的に表現
- 余白と情報階層を重視し、CTAを明確化

5. サイト構成（04-sitemap準拠）
- 公開サイト主要セクション:
  - トップ
  - 総合案内（ペットライフプラスとは/サービス詳細/料金説明/活用情報/ご利用までの案内）
  - 導入事例と実績
  - ログイン / Webアプリ接続
  - お問い合わせ
  - 運営情報・規約
- ワイヤーフレームの意図（ヒーロー→機能紹介→実績→CTA→フッター）を反映

6. GitHub Pages公開可能な構成
- ルーティング不要の静的構成
- 以下のようなファイル構成で作成:
  - index.html
  - concept.html
  - service.html
  - flow.html
  - faq.html
  - contact.html
  - css/style.css
  - js/main.js
  - img/...
- すべて相対パスでリンク
- ビルドツール不要（そのまま Pages へ配置可能）

【JavaScript要件（最低限）】
- ハンバーガーメニュー開閉
- スムーススクロール（同一ページ内リンク）
- FAQアコーディオン
- フォームの基本バリデーション（必須、メール形式）

【成果物】
- 完成したHTML/CSS/JSコード一式
- 各ファイルの内容をフルで出力
- 最後に「GitHub Pagesで公開する手順」を3〜5ステップで記載

【注意】
- ダミーテキストではなく、参照資料の文脈に沿った日本語コピーを入れる
- 可読性を最優先し、過剰なアニメーションは避ける
- Lighthouseのアクセシビリティを意識した実装にする
